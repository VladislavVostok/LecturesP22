
### 1. Приложение "Прогноз погоды" (Петрушин Богдан Станиславович)

**Описание:** Создать приложение, которое показывает погоду для выбранного города.

**Функционал:**
- Поле для поиска города.
- Взаимодействие с публичным API погоды (например, OpenWeatherMap, WeatherAPI).
- Отображение текущей погоды: температура, влажность, скорость ветра, иконка погоды.
- Показ прогноза на 5 дней вперед.
- Возможность сохранять несколько городов в "Избранное" (используя `localStorage`).
- (Опционально) Возможность переключать единицы измерения (Цельсий/Фаренгейт).

**Ключевые навыки:** `useState`, `useEffect`, работа с асинхронными запросами (API), условный рендеринг, работа с `localStorage`.

### 2. Поисковик рецептов

**Описание:** Приложение для поиска рецептов по ингредиентам или названию блюда.

**Функционал:**
- Поисковая строка.
- Использование API для рецептов (например, TheMealDB, Edamam).
- Отображение результатов поиска в виде сетки карточек с изображением и названием блюда.
- При клике на карточку — переход на страницу с детальным описанием рецепта (ингредиенты, шаги приготовления).
- (Опционально) Возможность добавлять ингредиенты из рецепта в "список покупок".

**Ключевые навыки:** `react-router-dom` для навигации, динамические маршруты, API, управление состоянием.

### 3. Канбан-доска (аналог Trello) (Богданов Даниил Романович)

**Описание:** Интерактивная доска с колонками для управления задачами.

**Функционал:**
- Создание колонок (например, "Сделать", "В процессе", "Готово").
- Добавление новых карточек-задач в любую колонку.
- Реализация перетаскивания (Drag-and-Drop) карточек между колонками.
- Сохранение состояния доски в `localStorage`, чтобы задачи не пропадали после перезагрузки страницы.
- (Опционально) Редактирование и удаление задач.

**Ключевые навыки:** Сложное управление состоянием, работа со сторонними библиотеками (например, `react-beautiful-dnd`), манипуляция с массивами данных.

### 4. Приложение-викторина (Quiz) (Басюк Артем Валерьевич)

**Описание:** Приложение для прохождения тестов на разные темы.

**Функционал:**
- Использование API для вопросов (например, Open Trivia Database).
- Пользователь выбирает категорию и сложность вопросов.
- Вопросы показываются по одному с вариантами ответов.
- После ответа пользователя показывается, был ли он правильным.
- В конце викторины выводится итоговый счет.

**Ключевые навыки:** Управление состоянием (текущий вопрос, счет), работа с API, логика игры.

### 5. Каталог фильмов (Климкин Глеб Александрович)

**Описание:** Приложение для просмотра информации о фильмах, похожее на IMDb или КиноПоиск.

**Функционал:**
- Использование API (например, The Movie Database (TMDB)).
- Отображение списков фильмов: "Популярные", "Топ-рейтинг", "Скоро в прокате".
- Поиск фильмов по названию.
- Динамическая страница для каждого фильма с детальной информацией: постер, описание, актерский состав, рейтинг.

**Ключевые навыки:** `react-router-dom` (вложенные и динамические маршруты), работа с API, пагинация (для длинных списков).

### 6. Простой интернет-магазин (витрина)

**Описание:** Фронтенд-часть небольшого интернет-магазина с корзиной.

**Фунционал:**
- Отображение сетки товаров, полученных из mock-API (можно создать `db.json` или использовать axios для получения данных с `dummyjson.com`).
- У каждого товара есть название, цена, изображение и кнопка "Добавить в корзину".
- Реализация корзины: пользователь может добавлять/удалять товары и видеть общую сумму заказа.
- Состояние корзины должно быть глобальным (доступным из разных компонентов).

**Ключевые навыки:** Глобальное управление состоянием (zustand), работа с массивами объектов, проектирование компонентной архитектуры.

### 7. Markdown-редактор (Ëдгаров Акрам Акмалович)

**Описание:** Приложение с двумя панелями для написания текста в формате Markdown и его предпросмотра в реальном времени.

**Функционал:**
- Левая панель — текстовое поле (`<textarea>`) для ввода Markdown-разметки.
- Правая панель — отображение отформатированного HTML-кода, который обновляется по мере ввода текста.
- Использование сторонней библиотеки (например, `marked` или `react-markdown`) для конвертации Markdown в HTML.

**Ключевые навыки:** Управление состоянием инпутов, интеграция сторонних библиотек, обработка событий.
